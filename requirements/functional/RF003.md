# RF003

## Caso de Uso

Gerenciar Endereço

## Descrição

O sistema deve permitir que usuários gerenciem seus endereços de entrega e os endereços da loja. Nesse caso, o gerenciar seria:
- Registrar;
- Editar;
- Detalhar.

## Atores

Usuário
 
## Dependências

[RF002](/requirements/functional/RF002.md)

## Cardinalidades

[ ADDRESS ] ( 0..1 ) ------------ ( 0..1 ) [ USER ]

Um endereço não precisa conter um usuário para ser registrado, nem um usuário precisa depender de um endereço para existir. Contudo, se houver algum relacionamento entre os dois, deve ser no máximo uma e somente uma relação.

[ ADDRESS ] ( 0..1 ) ------------ ( 0..1 ) [ STORE ]

Um endereço não precisa conter uma loja para ser registrado, nem uma loja precisa depender de um endereço para existir. Contudo, se houver algum relacionamento entre os dois, deve ser no máximo uma e somente uma relação.
